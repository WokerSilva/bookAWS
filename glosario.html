OnPremise

Infraestructura de TI que se encuentra físicamente en las instalaciones de una organización, en lugar de estar alojada en la nube. Requiere mantenimiento y gestión interna.

Cloud Computing

Modelo de computación que permite el acceso bajo demanda a recursos informáticos (como servidores, almacenamiento y aplicaciones) a través de Internet, sin necesidad de gestionar infraestructura física.

Data Lake

Repositorio centralizado que almacena grandes volúmenes de datos en su formato original (estructurados, semiestructurados y no estructurados), facilitando el análisis y la exploración.

Base de Datos

Sistema organizado para almacenar, gestionar y recuperar datos. Puede ser relacional (SQL) o no relacional (NoSQL), dependiendo de la estructura y el uso.

Gobernanza de Datos

Conjunto de políticas, procesos y estándares para garantizar la calidad, seguridad y uso adecuado de los datos en una organización.

SQL (Structured Query Language)

Lenguaje de programación utilizado para gestionar y manipular bases de datos relacionales. Permite consultar, insertar, actualizar y eliminar datos.

Migración a la Nube

Proceso de trasladar aplicaciones, datos y servicios desde una infraestructura OnPremise a un entorno de nube, como AWS, Azure o Google Cloud.

Escalabilidad en la Nube

Capacidad de un sistema en la nube para ajustar automáticamente sus recursos (como servidores o almacenamiento) en función de la demanda, sin interrupciones.

Data Warehouse

Almacén de datos optimizado para el análisis y la consulta de grandes volúmenes de información, generalmente utilizado en Business Intelligence.

DNS (Sistema de Nombres de Dominio)

Un sistema que traduce nombres de dominio legibles por humanos (como `www.ejemplo.com`) en direcciones IP numéricas, permitiendo que los dispositivos se comuniquen entre sí en Internet.

Red Virtual

Un entorno de red aislado lógicamente dentro de la nube, donde puedes lanzar recursos de AWS como instancias EC2. Permite control total sobre la configuración de red, como subredes y grupos de seguridad.

Archivo en Frío

Almacenamiento de datos optimizado para retención a largo plazo, con costos reducidos. Ideal para datos a los que se accede con poca frecuencia, como copias de seguridad o registros históricos.

Backups

Copias de seguridad de datos o sistemas que permiten la recuperación en caso de pérdida, corrupción o desastres. AWS ofrece servicios automatizados para gestionar y restaurar backups.

Balanceo de Carga

Distribución del tráfico de red entre múltiples servidores o instancias para mejorar la disponibilidad y escalabilidad de las aplicaciones.

Escalado Automático

Capacidad de ajustar automáticamente la cantidad de recursos informáticos (como instancias EC2) en función de la demanda, optimizando costos y rendimiento.

Serverless (Sin Servidor)

Un modelo de computación en el que el proveedor de la nube (AWS) gestiona automáticamente la infraestructura, permitiendo a los desarrolladores centrarse en el código sin preocuparse por servidores.

Edge Computing

Procesamiento de datos cerca de la fuente de generación (dispositivos IoT o usuarios finales) para reducir la latencia y mejorar la eficiencia.

Caché en Memoria

Almacenamiento temporal de datos en memoria RAM para acceder rápidamente a información frecuentemente solicitada, mejorando el rendimiento de las aplicaciones.

Base de Datos Relacional

Un tipo de base de datos que organiza los datos en tablas relacionadas entre sí, utilizando SQL para consultas y operaciones.

Base de Datos NoSQL

Base de datos diseñada para manejar grandes volúmenes de datos no estructurados o semiestructurados, sin necesidad de un esquema fijo.

Data Warehouse

Almacén de datos optimizado para el análisis y la consulta de grandes volúmenes de información, generalmente utilizado en Business Intelligence.

Series Temporales

Datos que se registran en intervalos de tiempo específicos, utilizados para análisis de tendencias, pronósticos y monitoreo.

Inferencia de ML

Proceso de utilizar un modelo de machine learning entrenado para hacer predicciones o tomar decisiones basadas en nuevos datos.

Computer Vision

Campo de la inteligencia artificial que permite a las máquinas interpretar y analizar imágenes y videos, como el reconocimiento facial o la detección de objetos.

NLP (Procesamiento de Lenguaje Natural)

Tecnología que permite a las máquinas entender, interpretar y generar lenguaje humano, utilizada en chatbots, análisis de sentimientos y traducción automática.

IoT (Internet de las Cosas)

Red de dispositivos físicos conectados a Internet que recopilan y comparten datos, como sensores, wearables y electrodomésticos inteligentes.

Blockchain

Tecnología de registro distribuido que permite la creación de registros inmutables y transparentes, utilizada en aplicaciones como contratos inteligentes y tokenización.

Contratos Inteligentes

Programas autoejecutables que se ejecutan en una blockchain y facilitan acuerdos automatizados entre partes sin intermediarios.

Trazabilidad

Capacidad de rastrear el origen y el historial de un producto o dato a lo largo de su ciclo de vida, común en cadenas de suministro y auditorías.

Geofencing

Técnica que utiliza GPS o RFID para crear límites virtuales alrededor de áreas geográficas, permitiendo acciones automatizadas cuando un dispositivo entra o sale de la zona.

Streaming de Datos

Transmisión continua de datos en tiempo real, utilizada en aplicaciones como análisis de métricas, monitoreo de sistemas y procesamiento de eventos.

API (Interfaz de Programación de Aplicaciones)

Conjunto de reglas y protocolos que permiten que diferentes aplicaciones se comuniquen entre sí, facilitando la integración de servicios.

CDN (Red de Entrega de Contenido)

Red distribuida de servidores que entrega contenido web (como imágenes, videos y scripts) de manera rápida y eficiente a usuarios finales.

VPN (Red Privada Virtual)

Conexión segura y cifrada entre redes locales y la nube, permitiendo acceso remoto seguro a recursos privados.

Enrutamiento

Proceso de dirigir el tráfico de red entre diferentes dispositivos o redes, optimizando la entrega de datos.

Subred

Segmento de una red IP que permite organizar y aislar recursos, mejorando la seguridad y el rendimiento.

Grupo de Seguridad

Conjunto de reglas de firewall que controlan el tráfico entrante y saliente de instancias o recursos en la nube.

Cifrado

Proceso de convertir datos en un formato ilegible para protegerlos de accesos no autorizados, utilizando algoritmos criptográficos.

Autenticación

Proceso de verificar la identidad de un usuario o dispositivo antes de permitir el acceso a un sistema o recurso.

Autorización

Proceso de determinar qué acciones o recursos puede acceder un usuario o aplicación después de la autenticación.

Escaneo de Vulnerabilidades

Evaluación automatizada de sistemas y aplicaciones para identificar y corregir posibles brechas de seguridad.

Auditoría

Revisión sistemática de sistemas, procesos o datos para garantizar el cumplimiento de normativas y estándares de seguridad.

Cumplimiento Normativo

Alineación de sistemas y procesos con regulaciones y estándares legales o industriales, como GDPR, HIPAA o PCI-DSS.

Data Lake

Repositorio centralizado que almacena grandes volúmenes de datos en su formato original, facilitando el análisis y la exploración.

ETL (Extracción, Transformación y Carga)

Proceso de extraer datos de múltiples fuentes, transformarlos en un formato útil y cargarlos en un sistema de destino, como un data warehouse.

Pipeline de Datos

Conjunto de procesos y herramientas que permiten el flujo de datos desde su origen hasta su destino, incluyendo transformaciones y validaciones.

Análisis de Sentimientos

Técnica de NLP que determina la actitud o emoción expresada en un texto, utilizada en redes sociales, reseñas y encuestas.

Pronóstico Predictivo

Uso de modelos de machine learning para predecir eventos futuros basados en datos históricos, como la demanda de productos o el clima.

Tokenización

Proceso de convertir activos físicos o digitales en tokens digitales que pueden ser gestionados y transferidos en una blockchain.

Microservicios

Arquitectura de software que divide una aplicación en servicios pequeños e independientes, cada uno con una función específica.

Contenedores

Tecnología que permite empaquetar una aplicación y sus dependencias en una unidad aislada, facilitando su despliegue y escalabilidad.

Orquestación de Contenedores

Gestión automatizada de contenedores, incluyendo su despliegue, escalado y monitoreo, utilizando herramientas como Kubernetes.

CI/CD (Integración Continua/Despliegue Continuo)

Prácticas de desarrollo que permiten la integración y entrega continua de código, automatizando pruebas y despliegues.

Monitoreo

Proceso de supervisar el rendimiento y la salud de sistemas y aplicaciones en tiempo real, utilizando métricas y alertas.

Logs

Registros de eventos generados por sistemas y aplicaciones, utilizados para diagnóstico, auditoría y análisis.

Escalado Horizontal

Aumento de la capacidad de un sistema añadiendo más instancias o nodos, en lugar de aumentar el poder de una sola instancia.

Escalado Vertical

Aumento de la capacidad de un sistema mejorando los recursos de una instancia existente, como CPU o memoria.

Alta Disponibilidad

Diseño de sistemas que garantiza un tiempo de actividad continuo, incluso en caso de fallos, mediante redundancia y replicación.

Recuperación ante Desastres

Conjunto de políticas y herramientas para restaurar sistemas y datos después de un fallo o desastre, minimizando el tiempo de inactividad.
